How does the internet work?

To begin with, there is a bunch of stuff I needed to understand, at least in a intuitive level. Things such as:

  • Client: Are the computers that want to enter on a website. These send a request to the servers via the internet, following the TCP/IP protocols.
  • Server: These computers store the data, the website, and receive the requests from the clients to access the website/data.
  • TCP/IP: These are a bunch of protocols that direct how to communicate between the computers in the web.
  • HTTP: This protocol helps out the render of a website. Since websites are built with HTML, which is an language for writing with hypertext, these two things work together. Is with this protocol that requests are send and (probably) the server's response to the client.
  • DNS: Kinda of topic, but still very useful for people like me, who forgets complex things. The servers and clients use an IP address, part of the Internet Protocol, so that the data knows where to travel. But IP address are hard to memorize, since is all just numbers. So, we use domains and names to refer to those IP address, which are easier to memorize.

With this knowledge, I'll try to explain how does the internet work. I learned it today tho, so don't quote me on that.

So, the client enters the URL of the website it wants to visit in the internet browser. Then, the browser checks in the DNS that the domain does refert to an IP address. If it does, it gets the information needed to send a request to the server of the domain searched. Such request is "written" with HTTP, and is sent through the internet following the TCP/IP.

The request is received by the website's server, which is basically asking for a copy of the website so that it can show in the browser of the client. If the server agrees to do so, it will send a response to the browser, kind of a confirmation that it will send a copy to the client's computer.

For sending the copy, it is divided into really tiny bits of information that will be sent to the the browser, each of them looking for the fastest way to do so. Those bits or data are called data packages, and is a strategy for allowing multiple users to download the website without clogging all the network.

The browser starts to receive the packages, and is its job to rebuild the website with those packages. After receiven all the data packages, the website is displayed in the browser and the client can watch whatever the heck it was looking for. All of these done in about one second.
